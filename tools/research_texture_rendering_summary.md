- Fabric's current rendering docs for 26.1.2 describe the post-1.21.6 render-state pipeline and warn that direct/raw GL style rendering is not the stable path.
- The repository already targets the new item render-state path by injecting after `ItemModelResolver.appendItemLayers` and appending an `ItemStackRenderState.LayerRenderState`.
- The static `assets/musicxcst/items/blueprint_cd.json` plus `models/item/blueprint_cd.json` can only select a static model and support component tinting; that cannot represent arbitrary per-stack 16x16 pixels by itself.
- The safest focused fix for this codebase is to keep the current render-state layer approach, make the appended layer behave like a real flat generated item layer, and verify via Gradle against the exact mapped 26.1.2 classes.
- `BuiltinItemRendererRegistry` examples from older Fabric versions are not used here because Minecraft 26.1.2 uses the newer extracted item render-state system, and this repo already has mixins against that system.
